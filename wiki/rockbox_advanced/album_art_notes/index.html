Album Art Tag Exploration

Initialisation

Album Art is drawn using two tags: %Cl to select an area and size, and %Cd to draw.

An Example would be the following:

%Cl(0,0,120,120)%Cd

Relative Viewports

Rockbox supports less rigid values for viewport dimensions. “-” makes a viewport’s width or height fill up the remaining screen space. Using negative values designates where the viewport will stop being drawn from the right or bottom side of the screen. Lastly percentage values allows a viewport to be drawn in relation to a device’s screen dimensions. Some examples of these in use:

# Full Screen Viewport.
%V(0,0,-,-,-)
#
# Viewport with a 10px margin around the screen.
%V(10,10,-10,-10,-) 
#
# Viewport drawn with percentages.
%V(2%,2%,90%,90%,-) 
#

Relative Viewports with Album Art

Using “-” or any negative values in the %Cl tag cause the album art to fail to render. This seems to happen regardless of wether or not the parent viewport is drawn with relative dimensions.

Using percentage values in %Cl draws the album art in relation to the values of the parent viewport.

Layers

Themes appear to be rendered from top to bottom. In this way, viewports called last render on top.
